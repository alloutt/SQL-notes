#parameter passing at runtime

myscript.sh
	#!/bin/bash
  dir_name=$1
  mkdir ~/dir_name
  echo $1 folder is created.

bash myscript.sh TEST      #"TEST" is parameter passed to $1 in script

output:TEST folder is created

similarly,u can pass $1,$2,$3,$4,... in script as 
bash script.sh var1 var2 var3 ...
$0 stored the filename by default

#if...else(conditional statements)
> -gt
>= -ge
< -lt
<= -le
== -eq

script.sh
	#!/bin/bash
	name=karan
	age=10
	if [ $age -gt 18 ]
	then
	echo "you are eligible for voting"
	else
	echo "you are not eligible for voting"
	fi
	echo "script completed"

-f #option-checks the file inside if statement in script

#!/bin/bash
if [ -f "/home/ubuntu/data.txt" ]
then
echo "file exist"
else
echo "file not exist"
fi 
echo "successful"
	
-d #option-checks the dir inside if statement in script

#!/bin/bash
if [ -d "/home/ubuntu/test1" ]
then
echo "dir exist"
else
mkdir /home/ubuntu/test1
fi 
echo "successful"

#running hive code from terminal
hive -e "hive_command"
OR
echo "hive_command" | hive

While loop:
	block of code running repeatedly
SYNTAX:
while command
do
code
done

Example:
while read -r table_name
do
echo $table_name table backup started
echo create table ${table_name}_bkp as select \* from $table_name \; | hive
echo $table_name table backup completed
done < table.txt

table.txt
t1
t2
t3
t4
t5

Sleep:
	wait for given sec
SYNATX:
echo something
sleep 5
echo something2

here it will wait for 5 sec before executing 2nd command

crontab:
	used to schedule script for particular time with particalar frequency

	min	hr	dayOfMonth	monthOfYear	dayofWeek
	*	*	*	*	*
	0-59	0-23	1-31	1-12 OR JAN-DEC	0-6 OR SUN-SAT

*	any value
,	value list separator
-	range of values
/	step values


https://crontab.guru/

HOW TO RUN:

crontab -e

	then paste the job in file like

	* * * * * myscript.sh
	save file and exit

Another tip:u can comment the job to not executing it.
