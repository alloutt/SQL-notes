HADOOP(introduced in 2007-08):(written in JAVA)
	initially google was facing challange for storage and process in around 2003-04,they team including person Doug Cutting came up with gfs(google file system)
for processing still was issue,they came up with MapReduce only on paper,later Doug Cuting joined Yahoo,there they implemented both storage and processing solution and named HDFS for storage,MapReduce:MR for processing.
later,Doug Cutting built an application named Hadoop.made it open-source(2007-08)
HADOOP is a OPEN SOURCE FRAMEWORK for CHALLANGES FOR STORAGE and PROCESSING DATA USING CLUSTER OF COMMODITY HARDWARE//COMMODITY HARDWARE means cheap hardware. 
 Deamons running-
  NameNode- on Master- only stores Metadata(info about other nodes,data about data is metadeta)
  DataNode- on slave respolsibility:send signal every 3 sec to NameNode that DataNode is working---every 10th signal beat(heartbeat) i.e. 30sec it sends block report to NameNode
DFS-service that divides data if space is not available block size ... hadoop 1.x 64mb,hadoop 2.x and 3.x = 128mb.default
when data is divided its fast to retrive too
Cluster is scalable.
HDFS is the combined storage of all nodes
NodeBalancer-balaces data stored in distrubuted manner

 hadoop fs -get <hdfs_absolute_path> <local_path>  //downloading file from HDFS to local
 
 hadoop fs -touchz /<filepath>        //creating file on HDFS
 
 hadoop fs -cp <src_hdfs_path> <desti_hdfs_path> //copy hdfs //can also copy folders
 
 hadoop fs -mv <src> <desti> //move,rename hdfs 
 
 hadoop fs -rm <path>  //remove file,-r option for recursive delete(situation: deleting folder) 
 similarly df,du,chmod,cat,head,tail...etc.
